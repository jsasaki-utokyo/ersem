# Carbonate Engine Test Harness

This directory contains a verification harness for the `carbonate_engine` module,
which provides a PyCO2SYS-compatible carbonate chemistry solver for ERSEM.

## Overview

The carbonate-engine solver (`src/carbonate_engine.F90`) computes pH and pCO2 from
DIC (dissolved inorganic carbon) and TA (total alkalinity) using robust root-finding
(Brent's method). This test harness validates the solver against reference values
generated by PyCO2SYS.

**The test driver calls the actual `carbonate_engine_solve` subroutine** (not a
reimplementation), using a minimal mock of `fabm_types` for standalone compilation.

## Files

- `test_driver.f90` - Fortran test driver that calls `carbonate_engine_solve` directly
- `fabm_types_mock.F90` - Minimal mock providing `rk` kind parameter
- `reference_cases.csv` - Reference test cases (T, S, DIC, TA -> pH, pCO2)
- `generate_reference.py` - Python script to regenerate reference cases using PyCO2SYS
- `Makefile` - Build and run tests
- `README.md` - This file

## Running the Tests

### Prerequisites

1. A Fortran compiler (gfortran recommended)

### Compile and Run

```bash
cd tests/carbonate_engine
make test
```

Or manually:
```bash
gfortran -O2 -fcheck=all -o test_driver \
    fabm_types_mock.F90 ../../src/carbonate_engine.F90 test_driver.f90
./test_driver reference_cases.csv
```

### Test Coverage

The test driver validates four categories:

1. **phscale=1 (Total scale)**: 20 cases against PyCO2SYS reference CSV
2. **phscale=0 (SWS scale)**: 5 cases checking Total/SWS pH offset consistency
3. **phscale=-1 (backward compat)**: 5 cases checking convergence and range
4. **Sub-zero temperature**: 2 cases for polar waters and legacy clamping

### Exit Codes

- `0` - All tests passed
- `1` - One or more tests failed, or no test cases found

## Regenerating Reference Cases

The reference cases are generated using PyCO2SYS. To regenerate:

```bash
pip install PyCO2SYS
python generate_reference.py
```

This will overwrite `reference_cases.csv` with fresh values from PyCO2SYS.

## PyCO2SYS Settings

The reference cases use the following PyCO2SYS settings to match the carbonate_engine
implementation with `opt_k_carbonic=1`:

| Setting | Value | Description |
|---------|-------|-------------|
| opt_k_carbonic | 4 | Lueker et al. (2000), total pH scale |
| opt_k_bisulfate | 1 | Dickson (1990) |
| opt_k_fluoride | 2 | Perez & Fraga (1987) |
| opt_total_borate | 2 | Lee et al. (2010) |
| opt_pH_scale | 1 | Total scale |

## Tolerances

Tests pass if the calculated values are within:

- **phscale=1**: pH ±1e-6, pCO2 ±0.01 µatm (tight PyCO2SYS match)
- **phscale=0**: Total-SWS pH difference in (0, 0.02), pCO2 ±0.5 µatm
- **phscale=-1**: pH in [6, 9], pCO2 in [50, 5000] µatm (sanity range)
